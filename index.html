Case 01 · 主打项目

CampRank 户外帐篷智能推荐系统

它不是普通商品列表，而是一个“输入需求 → 筛选商品 → 解释推荐原因 → 提醒购买风险”的决策辅助工具。

预算、露营场景、购买偏好，例如防水优先、性价比优先、售后稳定。

Step 2

系统匹配商品数据

比较价格、参数、售后、评论风险，并识别缺失信息。

Step 3

输出可解释推荐

给出首选、低价备选、谨慎选择，并说明为什么推荐。

CampRank recommendation flow

用户需求

预算¥500 - ¥1200
场景周末露营 / 防水优先
偏好性价比 + 售后稳定

推荐结果

首选推荐综合匹配高，风险较低
低价备选预算友好，核心需求满足
谨慎选择参数缺失或评论风险需确认

推荐依据

  • 价格是否落在预算内
  • 参数是否匹配使用场景
  • 售后和评论风险是否可接受
  • 缺失数据不会被当作满足条件

这个项目直观展示什么？

面试官打开后能看到：用户不是自己翻商品，而是通过系统获得有理由、有风险提示的购买建议。这体现的是推荐逻辑、产品判断和全栈实现能力。

我完成的工作

  • 设计推荐结果结构：首选、低价备选、谨慎选择。
  • 设计匹配规则：预算、场景、偏好、参数、售后、评论风险。
  • 实现缺失数据保守处理，避免“看起来都满足”的假推荐。
  • 搭建 React + FastAPI + SQLite 的前后端 Demo。

Case 02 · 数据自动化

GitHub Trending 开源趋势排行系统

自动抓取热门开源仓库，用评分算法生成趋势榜单，再通过静态页面展示。

代码
GitHub API抓取仓库数据
Python计算热度分数
Actions每 6 小时自动更新
Pages静态页面展示

解决的问题

热门项目不能只看 star 总数，还要考虑 fork、活跃时间和更新频率。这个项目把开源趋势发现做成自动化看板。

我做了什么

  • 用 GitHub REST API 获取多语言热门仓库。
  • 设计 stars + forks + 时间衰减的综合评分。
  • 用 GitHub Actions 定时刷新数据文件。
  • 用原生 HTML/CSS/JS 构建轻量趋势看板。

Case 03 · 前端交互基础

Gomoku Game 网页版五子棋

用 Canvas 和原生 JavaScript 实现完整棋盘交互、回合状态和胜负判断。

代码

解决的问题

通过一个完整的棋类游戏展示前端事件处理、状态管理和规则算法，而不是只做静态页面。

我做了什么

  • 使用 Canvas 绘制 15×15 棋盘和棋子。
  • 实现双人轮流落子、悔棋、重开和平局判断。
  • 实现横向、纵向、斜向五连胜检测。
  • 无依赖运行，适合展示原生 JavaScript 基础。
